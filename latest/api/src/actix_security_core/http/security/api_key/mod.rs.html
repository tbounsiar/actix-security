1//! API Key Authentication for Actix Web.
2//!
3//! # Overview
4//!
5//! API Key authentication is a simple authentication method where clients
6//! include a pre-shared key in their requests. It's commonly used for:
7//! - Service-to-service communication
8//! - Public APIs with usage tracking
9//! - Simple authentication without user sessions
10//!
11//! # Key Locations
12//!
13//! API keys can be extracted from:
14//! - **Header** (recommended): `X-API-Key: your-api-key`
15//! - **Authorization header**: `Authorization: ApiKey your-api-key`
16//! - **Query parameter**: `?api_key=your-api-key` (less secure)
17//!
18//! # Usage
19//!
20//! ## Basic Setup
21//!
22//! ```ignore
23//! use actix_security::http::security::api_key::{
24//!     ApiKeyAuthenticator, InMemoryApiKeyRepository, ApiKeyConfig, ApiKey,
25//! };
26//!
27//! // Create API key repository
28//! let mut repository = InMemoryApiKeyRepository::new();
29//!
30//! // Add API keys
31//! repository.add_key(ApiKey::new("sk_live_abc123")
32//!     .name("Production Key")
33//!     .roles(vec!["API_USER".into()])
34//!     .authorities(vec!["api:read".into(), "api:write".into()]));
35//!
36//! // Create authenticator
37//! let authenticator = ApiKeyAuthenticator::new(repository);
38//! ```
39//!
40//! ## With Custom Header
41//!
42//! ```ignore
43//! let config = ApiKeyConfig::header("Authorization")
44//!     .prefix("ApiKey ");  // Expects: Authorization: ApiKey sk_xxx
45//!
46//! let authenticator = ApiKeyAuthenticator::with_config(repository, config);
47//! ```
48//!
49//! ## Multiple Locations
50//!
51//! ```ignore
52//! let config = ApiKeyConfig::new()
53//!     .header("X-API-Key")
54//!     .query_param("api_key")
55//!     .authorization_scheme("ApiKey");
56//!
57//! let authenticator = ApiKeyAuthenticator::with_config(repository, config);
58//! ```
59//!
60//! # Spring Security Comparison
61//!
62//! | Spring Security | Actix Security |
63//! |-----------------|----------------|
64//! | Custom `AuthenticationFilter` | `ApiKeyAuthenticator` |
65//! | `AuthenticationProvider` | `ApiKeyRepository` |
66//! | `AbstractPreAuthenticatedProcessingFilter` | `ApiKeyConfig` locations |
67//!
68//! # Security Considerations
69//!
70//! 1. **Use HTTPS** - API keys are transmitted in plaintext
71//! 2. **Rotate keys** - Implement key rotation policies
72//! 3. **Limit scope** - Use authorities to restrict key capabilities
73//! 4. **Rate limit** - Prevent abuse with rate limiting per key
74//! 5. **Audit** - Log API key usage for security monitoring
75//!
76//! # Example with Middleware
77//!
78//! ```ignore
79//! use actix_security::http::security::{
80//!     SecurityTransform, AuthenticationManager,
81//!     api_key::{ApiKeyAuthenticator, InMemoryApiKeyRepository, ApiKey},
82//! };
83//!
84//! let mut repository = InMemoryApiKeyRepository::new();
85//! repository.add_key(ApiKey::new("sk_test_123").roles(vec!["USER".into()]));
86//!
87//! let authenticator = ApiKeyAuthenticator::new(repository);
88//!
89//! App::new()
90//!     .wrap(SecurityTransform::new()
91//!         .config_authenticator(move || authenticator.clone()))
92//!     .service(my_api_endpoint)
93//! ```
